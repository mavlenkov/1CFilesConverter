## ADDED Requirements

### Requirement: Загрузка расширения в информационную базу (ext2ib)
Скрипт ext2ib SHALL принимать путь к источнику расширения (CFE-файл, каталог XML, EDT-проект), путь к целевой ИБ и имя расширения. Скрипт SHALL загрузить расширение конфигурации в целевую ИБ.

#### Scenario: Загрузка из CFE-файла
- **WHEN** источник — файл с расширением `.cfe`, назначение — ИБ и задано имя расширения
- **THEN** скрипт SHALL загрузить расширение из CFE через Designer `/LoadCfg` с указанием `-Extension`

#### Scenario: Загрузка из каталога XML
- **WHEN** источник — каталог XML расширения и назначение — ИБ
- **THEN** скрипт SHALL загрузить расширение из XML через Designer `/LoadConfigFromFiles` с `-Extension`

#### Scenario: Загрузка из EDT-проекта
- **WHEN** источник — EDT-проект расширения и назначение — ИБ
- **THEN** скрипт SHALL экспортировать EDT в промежуточный XML, затем загрузить XML в ИБ как расширение

#### Scenario: Опциональное обновление базы данных
- **WHEN** переменная `V8_UPDATE_DB` установлена в `1`
- **THEN** скрипт SHALL после загрузки расширения выполнить обновление конфигурации БД

### Requirement: Выгрузка расширения в CFE-файл (ext2cfe)
Скрипт ext2cfe SHALL принимать путь к источнику расширения (ИБ, каталог XML, EDT-проект), имя расширения и путь к результирующему CFE-файлу.

#### Scenario: Выгрузка из ИБ в CFE
- **WHEN** источник — ИБ с загруженным расширением и назначение — путь к CFE-файлу
- **THEN** скрипт SHALL выгрузить расширение через Designer `/DumpCfg` с `-Extension`

#### Scenario: Выгрузка из XML в CFE
- **WHEN** источник — каталог XML расширения и назначение — путь к CFE-файлу
- **THEN** скрипт SHALL загрузить XML в промежуточную ИБ как расширение, затем выгрузить в CFE

#### Scenario: Выгрузка из EDT в CFE
- **WHEN** источник — EDT-проект расширения и назначение — путь к CFE-файлу
- **THEN** скрипт SHALL экспортировать EDT в XML, загрузить XML в промежуточную ИБ, затем выгрузить в CFE

### Requirement: Разбор расширения в XML (ext2xml)
Скрипт ext2xml SHALL принимать путь к источнику расширения (ИБ, CFE-файл, EDT-проект), имя расширения и путь к каталогу назначения XML.

#### Scenario: Разбор из ИБ в XML
- **WHEN** источник — ИБ с расширением и назначение — каталог
- **THEN** скрипт SHALL выгрузить расширение в XML через Designer `/DumpConfigToFiles` с `-Extension`

#### Scenario: Разбор из CFE в XML
- **WHEN** источник — CFE-файл и назначение — каталог
- **THEN** скрипт SHALL загрузить CFE в промежуточную ИБ, затем выгрузить расширение в XML

#### Scenario: Разбор из EDT в XML
- **WHEN** источник — EDT-проект расширения и назначение — каталог
- **THEN** скрипт SHALL экспортировать EDT в XML через ring/1cedtcli без промежуточной загрузки в ИБ. Блок LoadCfg SHALL НЕ выполняться для EDT-источника.

### Requirement: Конвертация расширения в EDT-проект (ext2edt)
Скрипт ext2edt SHALL принимать путь к источнику расширения (ИБ, CFE-файл, каталог XML), имя расширения и путь к каталогу EDT-проекта.

#### Scenario: Конвертация из XML в EDT
- **WHEN** источник — каталог XML расширения и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL подготовить базовую ИБ (если задана), но SHALL НЕ выполнять `/LoadCfg` на XML-каталог. Импорт в EDT SHALL выполняться напрямую через ring/1cedtcli.

#### Scenario: Конвертация из CFE в EDT
- **WHEN** источник — CFE-файл и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL загрузить CFE в промежуточную ИБ, выгрузить в промежуточный XML, затем импортировать в EDT

#### Scenario: Конвертация из ИБ в EDT
- **WHEN** источник — ИБ с расширением и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL выгрузить расширение ИБ в промежуточный XML, затем импортировать в EDT

### Requirement: Указание базовой ИБ и конфигурации для расширений
При работе с расширениями скрипты SHALL поддерживать указание базовой информационной базы через `V8_BASE_IB` и опциональную загрузку базовой конфигурации через `V8_BASE_CONFIG`.

#### Scenario: Базовая ИБ задана
- **WHEN** переменная `V8_BASE_IB` указывает на существующую ИБ
- **THEN** скрипт SHALL использовать указанную ИБ как рабочую базу для операций с расширением

#### Scenario: Базовая конфигурация задана
- **WHEN** переменная `V8_BASE_CONFIG` указывает на CF-файл или другой источник конфигурации
- **THEN** скрипт SHALL загрузить указанную конфигурацию в промежуточную ИБ через `conf2ib.sh` перед операциями с расширением

#### Scenario: Базовая ИБ не задана
- **WHEN** переменная `V8_BASE_IB` не задана
- **THEN** скрипт SHALL создать пустую временную ИБ для операций с расширением

### Requirement: Имя расширения
Все скрипты конвертации расширений SHALL принимать имя расширения как параметр.

#### Scenario: Имя расширения передано как третий параметр
- **WHEN** при вызове скрипта указан третий позиционный параметр (имя расширения)
- **THEN** скрипт SHALL использовать это имя для идентификации расширения в ИБ через параметр `-Extension`

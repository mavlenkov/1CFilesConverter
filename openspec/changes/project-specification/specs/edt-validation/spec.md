## ADDED Requirements

### Requirement: Валидация EDT-проекта (edt-validate)
Скрипт edt-validate SHALL принимать путь к источнику (EDT-проект, каталог XML конфигурации или расширения, CF-файл, CFE-файл, EPF/ERF-файлы, ИБ) и путь к файлу отчёта. Скрипт SHALL автоматически определить тип источника, при необходимости конвертировать в EDT-проект, и выполнить валидацию средствами EDT.

#### Scenario: Валидация EDT-проекта напрямую
- **WHEN** источник — EDT-проект (содержит `DT-INF`) и указан путь к файлу отчёта
- **THEN** скрипт SHALL запустить валидацию через ring/1cedtcli и сохранить отчёт в указанный файл

#### Scenario: Валидация из CF-файла
- **WHEN** источник — файл с расширением `.cf`
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `conf2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация из CFE-файла
- **WHEN** источник — файл с расширением `.cfe`
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `ext2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация из XML-каталога конфигурации
- **WHEN** источник — каталог, содержащий `Configuration.xml` без признака расширения (`<objectBelonging>`)
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `conf2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация из XML-каталога расширения
- **WHEN** источник — каталог, содержащий `Configuration.xml` с признаком расширения (`<objectBelonging>`)
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `ext2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация из файловой ИБ
- **WHEN** источник — каталог, содержащий `1Cv8.1CD`, или путь с префиксом `/F`
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `conf2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация из серверной ИБ
- **WHEN** путь источника начинается с `/S`
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `conf2edt.sh`, затем выполнить валидацию

#### Scenario: Валидация внешних обработок и отчётов
- **WHEN** источник — каталог, содержащий файлы `.epf`, `.erf` или XML-подкаталоги обработок
- **THEN** скрипт SHALL делегировать конвертацию в EDT через `dp2edt.sh`, затем выполнить валидацию

#### Scenario: Третий параметр — имя расширения
- **WHEN** задан третий позиционный параметр (имя расширения)
- **THEN** скрипт SHALL передать имя расширения в вызовы `ext2edt.sh`

### Requirement: Формат отчёта валидации
Скрипт edt-validate SHALL сохранить отчёт валидации, сформированный инструментом EDT, в указанный файл.

#### Scenario: Отчёт создаётся по указанному пути
- **WHEN** валидация завершена (успешно или с ошибками)
- **THEN** скрипт SHALL сохранить отчёт EDT-валидации в файл по пути, указанному вторым параметром

#### Scenario: Отсутствие инструментов EDT
- **WHEN** ни ring, ни 1cedtcli не обнаружены в системе и не заданы через переменные `RING_TOOL`/`EDTCLI_TOOL`
- **THEN** скрипт SHALL завершиться с ошибкой и сообщением о необходимости установки EDT

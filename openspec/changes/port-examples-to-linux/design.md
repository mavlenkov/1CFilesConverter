## Context

Проект уже имеет портированные `scripts/*.sh` и `tests/*.sh`. Паттерны портирования CMD→bash хорошо отработаны:
- `set VAR=value` → `VAR="value"`
- `%VAR%` → `"${VAR}"`
- `FOR /F` → `while read`
- `goto` → if/flags
- `tasklist/taskkill` → `pgrep/kill`
- `sqlcmd` → ветвление psql/sqlcmd по V8_DB_SRV_DBMS
- Пути: `%PROGRAMW6432%\1cv8\` → `/opt/1cv8/x86_64/`

## Goals / Non-Goals

**Goals:**
- Создать bash-эквиваленты всех 14 примеров
- Использовать загрузку .env аналогично существующим скриптам
- create_ib.sh: поддержка MSSQL (sqlcmd) и PostgreSQL (psql/createdb)

**Non-Goals:**
- Рефакторинг логики оригинальных скриптов
- Вынос общих функций в common.sh (примеры должны быть самодостаточными)

## Decisions

**1. Примеры самодостаточны** — не используют `source common.sh`, как и CMD-оригиналы. Каждый .sh содержит всю логику внутри себя.

**2. Загрузка .env** — встроена в каждый скрипт через стандартный паттерн `while IFS='=' read`.

**3. create_ib.sh** — вместо жёстких Windows-путей к .bak файлам, используется переменная для пути к бэкапу. Ветвление MSSQL/PostgreSQL через case по V8_DB_SRV_DBMS.

**4. Серверные подключения** — ibcmd-пути адаптированы для Linux (`/opt/1cv8/...`). Конструкция `Srvr=...;Ref=...;` (строка подключения 1С) не зависит от ОС.

## Risks / Trade-offs

- [Примеры содержат захардкоженные пути и параметры] → Это by design — примеры должны быть наглядными, пользователь адаптирует под свою среду.

## Context

При портировании CMD→bash скрипты ext2xml.sh и ext2edt.sh объединяют обработку нескольких типов источников в общие `if`-блоки с условиями вида `if [[ type == "cfe" ]] || [[ type == "edt" ]]`. В CMD-оригиналах ветвление реализовано через `goto`, что позволяет пропускать нерелевантные шаги (например, LoadCfg для XML-источника). При портировании через флаги и if-блоки эти пропуски были утеряны.

**ext2edt.sh**: блок `load_cfe` (строки 128–140) выполняет `/LoadCfg` для CFE и XML типов, хотя в CMD XML пропускает `load_cfe` через `goto export_xml`. В bash XML-источник ошибочно включён в условие на строке 98.

**ext2xml.sh**: блок `load_cfe` (строки 127–139) выполняет `/LoadCfg` для CFE и EDT типов. В CMD тоже есть такая же проблема, но для EDT вызов `LoadCfg` на каталог EDT бесполезен — далее запускается `run_edt_export`, который делает корректный экспорт.

## Goals / Non-Goals

**Goals:**
- Устранить ошибочный вызов `/LoadCfg` на XML-каталог в ext2edt.sh
- Устранить ошибочный вызов `/LoadCfg` на EDT-каталог в ext2xml.sh
- Сохранить корректную работу для CFE-источников

**Non-Goals:**
- Исправление аналогичных мест в CMD-скриптах
- Исправление заглушки EDT в dp2xml.sh (это не баг, а нереализованная функциональность)

## Decisions

### 1. ext2edt.sh: Вынести LoadCfg из общего блока с XML

**Решение:** Обернуть блок `load_cfe` (строки 128–140) в условие `if [[ "${V8_SRC_TYPE}" == "cfe" ]]`, чтобы LoadCfg выполнялся только для CFE-источника. XML-источник SHALL пропускать LoadCfg и переходить к export_ib → export_xml.

**Альтернатива:** Изменить условие на строке 98 с `cfe || xml` на только `cfe`. Отвергнута: XML тоже нуждается в `parse_base_ib` и `prepare_base_config` из этого блока, но не в LoadCfg.

### 2. ext2xml.sh: Вынести LoadCfg из общего блока с EDT

**Решение:** Обернуть блок `load_cfe` (строки 127–139) в условие `if [[ "${V8_SRC_TYPE}" == "cfe" ]]`. EDT-источник SHALL пропускать LoadCfg и сразу переходить к `run_edt_export`.

**Альтернатива:** Оставить как в CMD. Отвергнута: вызов `/LoadCfg` на каталог EDT — бесполезная операция, которая генерирует ошибку Designer.

## Risks / Trade-offs

- **[Расхождение с CMD]** ext2xml.sh будет отличаться от CMD-оригинала (CMD тоже вызывает LoadCfg для EDT). → Это сознательное улучшение, не ухудшение.
- **[Непротестированность]** ibcmd-ветка в блоке load_cfe тоже затронута. → Условие оборачивает весь блок целиком, включая обе ветки (designer/ibcmd).

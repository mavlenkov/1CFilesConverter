## Context

Репозиторий содержит скрипты двух типов: `.cmd` (Windows) и `.sh` (Linux/bash). Git по умолчанию (с `core.autocrlf=true`) конвертирует LF→CRLF при checkout на Windows, что ломает bash-скрипты. Файл `.gitattributes` фиксирует окончания строк на уровне репозитория, независимо от локальных настроек git.

## Goals / Non-Goals

**Goals:**
- Гарантировать LF-окончания для `.sh` файлов при любых настройках git
- Явно зафиксировать CRLF для `.cmd` файлов
- Нормализовать существующие файлы в репозитории

**Non-Goals:**
- Управление окончаниями строк для других типов файлов (.md, .env и т.д.)

## Decisions

**1. Минимальный `.gitattributes`** — только `.sh` и `.cmd` правила, без wildcard `* text=auto`.
- *Альтернатива*: `* text=auto` + переопределения — но это может затронуть .env, .md и другие файлы, поведение которых менять не нужно.

**2. `git add --renormalize .`** — одноразовая нормализация всех файлов после создания `.gitattributes`.
- Это создаст diff для файлов, у которых окончания строк в индексе не соответствуют новым правилам.

## Risks / Trade-offs

- [Большой diff при нормализации] → Ожидаемо, т.к. все .sh файлы будут перезаписаны с LF. Визуально diff будет пустой (только EOL-маркеры).

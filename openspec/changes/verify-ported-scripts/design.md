## Context

Окружение:
- **Локально**: 1C 8.3.27.1859 без ibcmd/EDT, нет лицензии
- **alcor** (Linux, SSH): 1C 8.3.27.1859 + ibcmd + ragent/ras + psql. EDT отсутствует
- **rigel** (Windows Server 2016 Datacenter): нужна установка OpenSSH Server (не встроен в 2016), после чего доступен по SSH для запуска CMD-тестов

## Goals / Non-Goals

**Goals:**
- Найти и исправить все ошибки портирования
- Объяснить каждый уровень тестирования и его назначение

**Non-Goals:**
- EDT-тесты (1cedtcli нет ни на одном сервере)
- EDT-тесты на Windows (1cedtcli предположительно отсутствует)

## Decisions

**Уровень 1: shellcheck (локально)**
Установить shellcheck, запустить на всех .sh. Shellcheck находит:
- Неэкранированные переменные в условиях
- Проблемы с word splitting
- Устаревшие конструкции
- Нерабочие паттерны

**Уровень 2: Код-ревью (параллельные агенты)**
Систематическое сравнение .sh с .cmd-оригиналами. Группы:
- scripts/*.sh (основная библиотека)
- tests/**/*.sh (тесты)
- examples/*.sh (примеры)
- wmscripts/*.sh (watchman)

**Уровень 3: Функциональные тесты на alcor**
- Склонировать репо на alcor
- Создать .env с параметрами окружения alcor
- Запустить test.sh — выполнит before/ → tests/ → after/
- EDT-тесты пропустятся (нет инструмента)

## Risks / Trade-offs

- [Лицензия 1С на alcor может отсутствовать] → Проверим перед запуском
- [Серверные тесты требуют ragent/ras] → Есть на alcor, но нужно проверить конфигурацию

## Why

При портировании тестов на Linux все 83 .sh файла были созданы с CRLF-окончаниями строк из-за настройки git `core.autocrlf`. Bash не может выполнять скрипты с CRLF, что приводит к синтаксическим ошибкам. Нужно добавить `.gitattributes`, чтобы гарантировать LF-окончания для .sh файлов независимо от настроек git пользователя.

## What Changes

- Создать файл `.gitattributes` в корне репозитория
- Указать `eol=lf` для всех `.sh` файлов
- Указать `eol=crlf` для всех `.cmd` файлов (явная фиксация текущего поведения)
- Нормализовать существующие файлы в репозитории через `git add --renormalize .`

## Impact

- Новый файл `.gitattributes` в корне репозитория
- Все `.sh` файлы в репозитории будут храниться и извлекаться с LF-окончаниями
- Все `.cmd` файлы сохранят CRLF-окончания
- Изменение затронет `scripts/*.sh`, `tests/**/*.sh` и любые будущие .sh файлы

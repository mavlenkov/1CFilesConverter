## Why

Проект содержит 84 CMD-теста (test.cmd + 8 before + 76 tests + 1 after), но Linux bash-порт скриптов конвертации не имеет тестов. Невозможно верифицировать корректность bash-скриптов без запуска тестов на Linux. Портирование тестов позволит проверять работоспособность bash-скриптов и обнаруживать регрессии.

## What Changes

- Портирование test.cmd → test.sh (фреймворк тестового запуска)
- Портирование 8 before-скриптов подготовки тестовых данных (*.cmd → *.sh)
- Портирование 76 тестовых скриптов (*.cmd → *.sh)
- Портирование 1 after-скрипта очистки (*.cmd → *.sh)
- Адаптация серверных тестов: psql вместо sqlcmd, ps/kill вместо tasklist/taskkill, Linux-пути RAC
- Адаптация теста 127 (designer_running): pgrep/kill вместо tasklist/taskkill, headless-запуск 1cv8
- Добавление tests/example.env для Linux (Linux-пути и значения по умолчанию)

## Impact

- `tests/test.sh` — новый файл, фреймворк запуска
- `tests/before/*.sh` — 8 новых файлов подготовки
- `tests/tests/*.sh` — 76 новых тестовых скриптов
- `tests/after/*.sh` — 1 новый файл очистки
- `tests/example.env` — обновление/дополнение для Linux
- Не затрагивает основные скрипты конвертации

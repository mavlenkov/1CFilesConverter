## ADDED Requirements

### Requirement: Конвертация конфигурации в информационную базу (conf2ib)
Скрипт conf2ib SHALL принимать путь к источнику конфигурации (CF-файл, каталог XML или EDT-проект) и путь к целевой информационной базе. Скрипт SHALL автоматически определить формат источника и выполнить необходимую цепочку конвертаций для загрузки конфигурации в целевую ИБ. Информационная база как источник не поддерживается.

#### Scenario: Загрузка из CF-файла в файловую ИБ
- **WHEN** источник — файл с расширением `.cf` и назначение — путь к файловой ИБ
- **THEN** скрипт SHALL загрузить конфигурацию из CF-файла в указанную ИБ через Designer `/LoadCfg`

#### Scenario: Загрузка из каталога XML в файловую ИБ
- **WHEN** источник — каталог, содержащий XML-файлы конфигурации, и назначение — путь к файловой ИБ
- **THEN** скрипт SHALL загрузить конфигурацию из XML в указанную ИБ через Designer `/LoadConfigFromFiles`

#### Scenario: Загрузка из EDT-проекта в файловую ИБ
- **WHEN** источник — каталог EDT-проекта (содержит `DT-INF`) и назначение — путь к файловой ИБ
- **THEN** скрипт SHALL сначала экспортировать EDT-проект в промежуточный XML, затем загрузить XML в ИБ

#### Scenario: Создание новой ИБ
- **WHEN** третий параметр равен `create` или переменная `V8_IB_CREATE` установлена в `1`
- **THEN** скрипт SHALL создать новую файловую ИБ через `CREATEINFOBASE` и загрузить в неё конфигурацию

#### Scenario: Опциональное обновление базы данных
- **WHEN** переменная окружения `V8_UPDATE_DB` установлена в `1`
- **THEN** скрипт SHALL после загрузки конфигурации выполнить обновление конфигурации БД через Designer `/UpdateDBCfg`

### Requirement: Конвертация конфигурации в CF-файл (conf2cf)
Скрипт conf2cf SHALL принимать путь к источнику конфигурации (ИБ, каталог XML, EDT-проект) и путь к результирующему CF-файлу. Скрипт SHALL автоматически определить формат источника и выгрузить конфигурацию в бинарный CF-файл.

#### Scenario: Выгрузка из ИБ в CF
- **WHEN** источник — информационная база и назначение — путь к CF-файлу
- **THEN** скрипт SHALL выгрузить конфигурацию из ИБ через Designer `/DumpCfg`

#### Scenario: Выгрузка из XML в CF
- **WHEN** источник — каталог XML и назначение — путь к CF-файлу
- **THEN** скрипт SHALL загрузить XML в промежуточную ИБ, затем выгрузить конфигурацию в CF-файл

#### Scenario: Выгрузка из EDT в CF
- **WHEN** источник — EDT-проект и назначение — путь к CF-файлу
- **THEN** скрипт SHALL экспортировать EDT в промежуточный XML, загрузить XML в промежуточную ИБ, затем выгрузить в CF

### Requirement: Конвертация конфигурации в XML (conf2xml)
Скрипт conf2xml SHALL принимать путь к источнику конфигурации (CF-файл, ИБ, EDT-проект) и путь к каталогу назначения XML. Скрипт SHALL разобрать конфигурацию в формат XML (формат Конфигуратора 1С).

#### Scenario: Разбор CF-файла в XML
- **WHEN** источник — CF-файл и назначение — каталог
- **THEN** скрипт SHALL загрузить CF в промежуточную ИБ, затем выгрузить в XML через Designer `/DumpConfigToFiles`

#### Scenario: Разбор ИБ в XML
- **WHEN** источник — информационная база и назначение — каталог
- **THEN** скрипт SHALL выгрузить конфигурацию ИБ в XML через Designer `/DumpConfigToFiles`

#### Scenario: Разбор EDT-проекта в XML
- **WHEN** источник — EDT-проект и назначение — каталог
- **THEN** скрипт SHALL экспортировать EDT в XML через ring/1cedtcli

#### Scenario: Очистка каталога назначения
- **WHEN** переменная `V8_CONF_CLEAN_DST` установлена в `1`
- **THEN** скрипт SHALL очистить каталог назначения перед выгрузкой XML

### Requirement: Конвертация конфигурации в EDT-проект (conf2edt)
Скрипт conf2edt SHALL принимать путь к источнику конфигурации (CF-файл, ИБ, каталог XML) и путь к каталогу EDT-проекта. Скрипт SHALL создать EDT-проект из исходной конфигурации.

#### Scenario: Конвертация из XML в EDT
- **WHEN** источник — каталог XML и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL импортировать XML в EDT-проект через ring/1cedtcli

#### Scenario: Конвертация из CF в EDT
- **WHEN** источник — CF-файл и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL загрузить CF в промежуточную ИБ, выгрузить в промежуточный XML, затем импортировать в EDT

#### Scenario: Конвертация из ИБ в EDT
- **WHEN** источник — информационная база и назначение — каталог EDT-проекта
- **THEN** скрипт SHALL выгрузить конфигурацию ИБ в промежуточный XML, затем импортировать XML в EDT

#### Scenario: Очистка каталога назначения EDT
- **WHEN** переменная `V8_CONF_CLEAN_DST` установлена в `1`
- **THEN** скрипт SHALL очистить каталог назначения перед созданием EDT-проекта

### Requirement: Автоматическое определение формата источника
Все скрипты конвертации конфигураций SHALL автоматически определять формат источника по его содержимому и структуре.

#### Scenario: Определение CF-файла
- **WHEN** путь источника указывает на файл с расширением `.cf`
- **THEN** скрипт SHALL распознать источник как бинарный CF-файл

#### Scenario: Определение файловой ИБ
- **WHEN** путь источника указывает на каталог, содержащий файл `1Cv8.1CD` (case-insensitive на Linux)
- **THEN** скрипт SHALL распознать источник как файловую информационную базу

#### Scenario: Определение серверной ИБ
- **WHEN** путь источника начинается с `/S` (case-insensitive)
- **THEN** скрипт SHALL распознать источник как серверную информационную базу

#### Scenario: Определение каталога XML
- **WHEN** путь источника указывает на каталог, содержащий файл `Configuration.xml`
- **THEN** скрипт SHALL распознать источник как каталог XML конфигурации

### Requirement: Параметры и переменные окружения конвертации конфигураций
Скрипты конвертации конфигураций SHALL поддерживать конфигурирование через переменные окружения.

#### Scenario: Указание инструмента конвертации
- **WHEN** переменная `V8_CONVERT_TOOL` установлена в `designer` или `ibcmd`
- **THEN** скрипт SHALL использовать указанный инструмент для выполнения конвертации

#### Scenario: Указание версии платформы
- **WHEN** переменная `V8_VERSION` задана
- **THEN** скрипт SHALL использовать указанную версию платформы 1С для поиска инструментов

#### Scenario: Авторизация в ИБ
- **WHEN** переменные `V8_IB_USER` и/или `V8_IB_PWD` заданы
- **THEN** скрипт SHALL передать учётные данные при подключении к ИБ через параметры Designer `/N` и `/P`

## ADDED Requirements

### Requirement: Разбор отчётов и обработок в XML (dp2xml)
Скрипт dp2xml SHALL принимать путь к источнику (каталог с EPF/ERF файлами) и путь к каталогу назначения. Скрипт SHALL разобрать каждый файл обработки/отчёта в формат XML. EDT-проект как источник распознаётся, но конвертация не реализована (заглушка).

#### Scenario: Разбор EPF/ERF файлов в XML
- **WHEN** источник — каталог, содержащий файлы `.epf` и/или `.erf`
- **THEN** скрипт SHALL для каждого файла создать подкаталог в назначении и выгрузить XML через Designer `/DumpExternalDataProcessorOrReportToFiles`

#### Scenario: Источник — единичный EPF/ERF файл
- **WHEN** источник — путь к одному файлу `.epf` или `.erf`
- **THEN** скрипт SHALL разобрать этот файл в XML в каталоге назначения

#### Scenario: EDT-проект как источник (заглушка)
- **WHEN** источник — EDT-проект (содержит `DT-INF`)
- **THEN** скрипт SHALL вывести информационное сообщение и завершиться без конвертации

#### Scenario: Неподдерживаемый тип источника
- **WHEN** источник не содержит EPF/ERF файлов и не является EDT-проектом
- **THEN** скрипт SHALL завершиться с ошибкой `[ERROR] Wrong path`

### Requirement: Сборка отчётов и обработок из XML (dp2epf)
Скрипт dp2epf SHALL принимать путь к источнику (каталог XML или EDT-проект) и путь к каталогу назначения. Скрипт SHALL собрать бинарные EPF/ERF файлы из исходных текстов.

#### Scenario: Сборка из XML в EPF/ERF
- **WHEN** источник — каталог XML обработок и назначение — каталог для EPF/ERF
- **THEN** скрипт SHALL для каждого подкаталога XML собрать бинарный файл через Designer `/LoadExternalDataProcessorOrReportFromFiles`

#### Scenario: Сборка из EDT в EPF/ERF
- **WHEN** источник — EDT-проект обработок и назначение — каталог для EPF/ERF
- **THEN** скрипт SHALL экспортировать EDT в промежуточный XML через ring/1cedtcli, затем собрать EPF/ERF из промежуточного XML через Designer

### Requirement: Конвертация отчётов и обработок в EDT (dp2edt)
Скрипт dp2edt SHALL принимать путь к источнику (каталог с EPF/ERF, каталог XML) и путь к каталогу EDT-проекта. Скрипт SHALL создать EDT-проект из исходных обработок.

#### Scenario: Конвертация из EPF/ERF в EDT
- **WHEN** источник — каталог с EPF/ERF файлами и назначение — каталог EDT
- **THEN** скрипт SHALL разобрать EPF/ERF в промежуточный XML через Designer, затем импортировать XML в EDT через ring/1cedtcli

#### Scenario: Конвертация из XML в EDT
- **WHEN** источник — каталог XML обработок и назначение — каталог EDT
- **THEN** скрипт SHALL импортировать XML в EDT через ring/1cedtcli

### Requirement: Обнаружение формата файлов обработок
Скрипты dp2xml и dp2edt SHALL автоматически определять формат источника по содержимому каталога.

#### Scenario: Каталог содержит EPF/ERF файлы
- **WHEN** в каталоге-источнике обнаружены файлы с расширениями `.epf` или `.erf` (glob-паттерн `*.epf`/`*.erf`, чувствителен к регистру)
- **THEN** скрипт SHALL распознать источник как бинарные обработки/отчёты

#### Scenario: Единичный EPF/ERF файл как источник
- **WHEN** путь источника указывает на файл с расширением `.epf` или `.erf` (case-insensitive)
- **THEN** скрипт SHALL распознать источник как единичный бинарный файл обработки/отчёта

### Requirement: Базовая конфигурация для обработок
При работе с обработками, которые зависят от базовой конфигурации, скрипты SHALL поддерживать указание базовой ИБ.

#### Scenario: Базовая ИБ задана
- **WHEN** переменная `V8_BASE_IB` указывает на базовую ИБ
- **THEN** скрипт SHALL использовать эту ИБ для операций Designer вместо пустой промежуточной ИБ
